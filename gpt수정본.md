# 컴퓨터구조 · 운영체제 핵심 요약

## Chapter 02 데이터

### 02-1 숫자 표현
- **비트(bit)**: 0과 1을 표현하는 가장 작은 정보 단위
- **바이트(byte)**: 8비트
- **이진법**: 0과 1만으로 수를 표현
- **십육진법**: 이진수를 짧게 표현하기 위해 사용
- 컴퓨터는 결국 모든 데이터를 **0과 1의 조합**으로 처리한다.

### 02-2 문자 표현
- **문자 집합(Character Set)**: 어떤 문자를 표현할지 정한 표
- **인코딩(Encoding)**: 문자를 0과 1로 바꾸는 규칙
- **ASCII**: 영문/기본 기호 중심
- **EUC-KR**: 한글 표현 가능, 범위 제한적
- **유니코드(Unicode)**: 다양한 언어를 통합 표현
- **UTF-8**: 유니코드를 가변 길이로 인코딩하는 방식

## Chapter 03 명령어

### 03-1 소스 코드와 명령어
- **고급 언어**: 사람이 이해하기 쉬운 언어
- **저급 언어**: 컴퓨터가 가까이 이해하는 언어
- **컴파일 언어**: 실행 전 전체 번역
- **인터프리터 언어**: 실행하면서 한 줄씩 번역
- **목적 파일**: 번역은 되었지만 바로 실행 불가할 수 있음
- **실행 파일**: 실제 실행 가능한 파일

### 03-2 명령어 구조
- 명령어는 보통 **연산 코드(opcode)** 와 **오퍼랜드(operand)** 로 구성
- **연산 코드**: 무엇을 할지
- **오퍼랜드**: 어떤 데이터를 대상으로 할지
- **주소 지정 방식**: 오퍼랜드를 어디서 가져올지 정하는 방법
  - 즉시 주소 지정
  - 직접 주소 지정
  - 간접 주소 지정
  - 레지스터 주소 지정

## Chapter 04 CPU와 작동 원리

### 04-1 ALU와 제어장치
- **ALU**: 산술/논리 연산 수행
- **제어장치**: 명령어 해석 후 제어 신호 발생
- CPU는 **연산장치 + 제어장치 + 레지스터** 중심으로 동작한다.

### 04-2 레지스터
- **프로그램 카운터(PC)**: 다음에 실행할 명령어 주소
- **명령어 레지스터(IR)**: 현재 실행 중인 명령어
- **메모리 주소 레지스터(MAR)**: 접근할 메모리 주소
- **메모리 버퍼 레지스터(MBR/MDR)**: 메모리와 주고받는 데이터
- **스택 포인터(SP)**: 스택의 꼭대기 위치
- **베이스 레지스터**: 기준 주소 저장

### 주소 지정 방식과 레지스터
- **스택 주소 지정 방식**: 스택 포인터 활용
- **변위 주소 지정 방식**: 기준 주소 + 변위값으로 접근

### 04-3 명령어 사이클과 인터럽트
- **명령어 사이클**: 인출 → 해석 → 실행
- CPU는 명령어를 반복적으로 읽고 실행한다.
- **인터럽트**: CPU가 하던 일을 잠시 멈추고 다른 작업을 먼저 처리하게 하는 신호
- **예외(Exception)**: 실행 중인 명령어 때문에 발생하는 특수 상황
  - 예: 0으로 나누기, 허용되지 않은 메모리 접근

## Chapter 05 CPU 성능 향상 기법

### 05-1 빠른 CPU를 위한 설계
- **클럭**: CPU 동작 속도의 기준
- **코어**: 명령어를 실행하는 핵심 처리 장치
- **멀티코어**: 여러 코어를 사용해 병렬 처리
- **하드웨어 스레드**: 한 코어가 여러 실행 흐름을 다루는 기술

### 05-2 명령어 병렬 처리
- **명령어 파이프라인**: 여러 명령어를 겹쳐 처리
- **슈퍼스칼라**: 한 번에 여러 명령어 실행
- **비순차적 명령어 처리**: 순서를 일부 바꿔 효율적으로 실행

### 05-3 CISC와 RISC
- **CISC**: 복잡한 명령어 제공, 명령어 수 적게
- **RISC**: 단순한 명령어 중심, 빠르고 규칙적
- 현대 CPU는 두 특성을 혼합적으로 활용하기도 한다.

## Chapter 06 메모리와 캐시 메모리

### 06-1 RAM 특징과 종류
- **RAM**: 휘발성, 빠른 작업용 메모리
- RAM 용량이 부족하면 **스왑** 때문에 성능 저하 가능
- **DRAM**: 저렴, 고집적, 주기억장치에 주로 사용
- **SRAM**: 빠르지만 비쌈, 캐시에 주로 사용
- **SDRAM**: 클럭에 맞춰 동작
- **DDR SDRAM**: 더 높은 전송률 제공

### 06-2 메모리 주소 공간
- **논리 주소**: CPU가 보는 주소
- **물리 주소**: 실제 메모리 주소
- **MMU**: 논리 주소를 물리 주소로 변환
- **메모리 보호**: 다른 프로세스/운영체제 영역 침범 방지

### 06-3 캐시 메모리
- **저장 장치 계층 구조**
  - 레지스터 → 캐시 → 메모리 → 보조기억장치
- 위로 갈수록 빠르고 작고 비쌈
- 아래로 갈수록 느리고 크고 저렴
- **캐시 메모리**: CPU와 메모리 속도 차이 완화
- **참조 지역성**
  - 시간 지역성: 최근 사용한 데이터 재사용
  - 공간 지역성: 근처 데이터 재사용

## Chapter 07 보조기억장치

### 07-1 다양한 보조기억장치
- **하드디스크(HDD)**: 자기적 방식 저장
- **플래시 메모리(SSD 등)**: 반도체 기반 저장장치
- 플래시 메모리는 **셀** 단위로 데이터 저장
- **SLC / MLC / TLC**: 셀당 저장 비트 수 차이
- TLC는 고집적에 유리하지만 속도·내구성은 상대적으로 불리

### 플래시 메모리의 단위
- **페이지(Page)**: 읽기/쓰기 단위
- **블록(Block)**: 삭제 단위
- 읽기/쓰기/삭제 단위가 다르다는 점이 특징이다.

### 07-2 RAID
- 여러 저장장치를 하나처럼 묶는 기술
- **RAID 0**: 성능 향상, 안정성 낮음
- **RAID 1**: 미러링, 안정성 높음
- **RAID 4**: 전용 패리티 디스크 사용
- **RAID 5**: 패리티 분산 저장
- **RAID 6**: 이중 패리티, 디스크 2개 장애 허용 가능

## Chapter 08 입출력장치

### 08-1 장치 컨트롤러와 드라이버
- **장치 컨트롤러**: CPU와 장치 사이 중개
- **장치 드라이버**: 운영체제가 장치를 제어하게 하는 소프트웨어
- **버퍼**: 속도 차이를 완화하기 위한 임시 저장 공간

### 08-2 입출력 방법
- **프로그램 입출력**: CPU가 직접 장치를 확인
- **메모리 맵 입출력**: 메모리 주소 공간과 장치 주소 공간을 함께 사용
- **고립형 입출력**: 메모리와 장치 주소 공간 분리
- **인터럽트 기반 입출력**: 장치가 CPU에게 완료를 알림
- **인터럽트 벡터**: 어떤 처리 루틴으로 갈지 알려줌
- **DMA 입출력**: CPU를 거치지 않고 장치가 메모리와 직접 데이터 전송

## Chapter 09 운영체제 시작하기

### 09-1 운영체제란
- 운영체제는 **하드웨어 자원을 관리하고 프로그램 실행 환경을 제공하는 시스템 소프트웨어**이다.

### 09-2 운영체제의 큰 그림
- **커널**: 운영체제의 핵심 부분
- **이중 모드**
  - 사용자 모드
  - 커널 모드
- **시스템 호출(System Call)**: 사용자 프로그램이 운영체제 기능을 요청하는 방법
- 핵심 서비스
  - 프로세스 관리
  - 메모리 관리
  - 파일 시스템 관리
  - 입출력 관리
  - 보호와 보안
- **가상 머신**: 하나의 물리 시스템 위에 여러 가상 환경 제공

## Chapter 10 프로세스와 스레드

### 10-1 프로세스 개요
- **프로세스**: 실행 중인 프로그램
- 운영체제는 PCB로 프로세스를 관리
- **문맥 교환(Context Switching)**: 실행 정보 저장/복원 후 다른 프로세스로 전환
- 프로세스 메모리 영역
  - 코드
  - 데이터
  - 힙
  - 스택

### 10-2 프로세스 상태와 계층 구조
- 상태: 생성 → 준비 → 실행 → 대기 → 종료
- **부모 프로세스 / 자식 프로세스** 구조 가능
- **fork()**: 자식 프로세스 생성
- **exec()**: 현재 프로세스 메모리 공간을 새 프로그램으로 교체

### 10-3 스레드
- **스레드**: 프로세스 내부 실행 흐름 단위
- 같은 프로세스의 스레드는 코드/데이터/힙 공유
- 스레드마다 스택, 레지스터는 별도
- **멀티프로세스**: 자원 분리, 안정성 높음
- **멀티스레드**: 자원 공유, 효율 높음

## Chapter 11 CPU 스케줄링

### 11-1 스케줄링 개요
- CPU는 한정된 자원이므로 스케줄링 필요
- **우선순위**에 따라 CPU 배분 가능
- **스케줄링 큐**
  - 준비 큐
  - 대기 큐
- **선점형 스케줄링**: 운영체제가 CPU 회수 가능
- **비선점형 스케줄링**: 스스로 반납할 때까지 사용

### 11-2 스케줄링 알고리즘
- **FCFS**: 먼저 온 순서대로 처리
- **SJF**: 짧은 작업 우선
- **RR**: 일정 시간씩 번갈아 처리
- **SRT**: 남은 시간이 가장 짧은 작업 우선
- **우선순위 스케줄링**: 높은 우선순위 우선
- **에이징**: 기아 현상 완화
- **다단계 큐**
- **다단계 피드백 큐**

### 추가 개념
- **CPU burst**: CPU를 집중적으로 사용하는 구간
- **I/O burst**: 입출력 대기/처리 구간

## Chapter 12 프로세스 동기화

### 12-1 동기화란
- 여러 프로세스/스레드가 **공유 자원**에 안전하게 접근하도록 실행 순서를 맞추는 것
- **공유 자원**: 여러 실행 흐름이 함께 사용하는 자원
- **임계 구역(Critical Section)**: 동시에 접근하면 문제가 생길 수 있는 영역
- **경쟁 상태(Race Condition)**: 실행 순서에 따라 결과가 달라지는 문제
- 대표 예시: **생산자와 소비자 문제**

### 12-2 동기화 도구
- **뮤텍스 락(Mutex Lock)**: 한 번에 하나만 접근 가능
- **세마포(Semaphore)**: 허용 가능한 접근 수를 카운트로 관리
- **모니터(Monitor)**: 공유 자원 접근과 대기를 고수준으로 관리하는 구조

## 동기 / 비동기 처리

- **동기 처리**: 작업이 끝날 때까지 기다림
- **비동기 처리**: 작업을 맡겨두고 다음 작업 진행
- 이는 **동기화**와 다른 개념이다.
  - 동기/비동기 = 작업 완료를 기다리는 방식
  - 동기화 = 공유 자원 접근 순서를 맞추는 것
